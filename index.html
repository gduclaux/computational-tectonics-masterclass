2-Day Masterclass · Geodynamics & Numerical Modelling

Introduction to
Computational Tectonics

A hands-on intensive course for PhD candidates and researchers: use Underworld2 to forward-model the thermal and tectonic evolution of lithospheric systems, and test scientific hypotheses numerically. This course is supported by UWA's International Advanced Studies' (IAS) Gledden Visting Fellowship.

// Learning Outcomes

What you will gain

By the end of the masterclass most of you will (hopefully) be able to frame a geological question as a simple numerical forward model, implement it in 2D in Underworld2, and extract science from the output.

🧭
Familiarise with forward numerical modelling

Understand the particle-in-cell finite element approach and how Underworld2 bridges Python scripting with HPC-grade computations.

💡
Conceptualise testable tectonic problems

Translate a geological hypothesis into a well-posed model: choose the right physics, select appropriate governing equations, and define meaningful boundary conditions.

🔨
Design and run simple models

Build a 2D crustal or lithospheric model from scratch — define geometry, material properties, rheology, and numerical solvers — then run it and iterate.

📊
Visualise and analyse results

Use ParaView for post-processing, ImageJ for image analysis, and Python for quantitative diagnostic metrics (e.g. extract PTt path).

// Before You Arrive

Prerequisites

No prior modelling experience is required, but the following will ensure you get the most out of the two days.

  • Geological curiosity Basic familiarity with rock mechanics, plate tectonics, lithospheric structure, and the geological questions you'd like to explore numerically.
  • Physics & maths basics Comfortable with vectors, derivatives, and the concept of conservation laws (mass, momentum, energy).
  • Computer literacy Able to navigate the file system, open a terminal, and run basic Unix/shell commands (cd, ls, mkdir).
  • Python fundamentals Python 3 installed; familiar with variables, loops, functions, and importing libraries. NumPy & matplotlib experience is a bonus.
💻 Install before (or during if your silly teacher didn't tell you early enough) Day 1

Please install Docker Desktop, ParaView, and ImageJ before arriving. The UW2 Docker image will be distributed on a USB stick to save bandwidth on Day 1 morning. Python 3 with NumPy, h5py, and matplotlib must be available on your machine.

👀 What a UW2 model looks like

A minimal but complete Stokes + energy model skeleton:

Python · underworld2 
# Enable autocompletion with TAB key in Jupyter notebook
%config Completer.use_jedi = False    
        
# 1. Load UW packages
from underworld import UWGeodynamics as GEO
from underworld import visualisation as vis
import numpy as np;

# 2. Set Scaling rules for non-dimensionalisation
u = GEO.UnitRegistry
velocity = 1 * u.centimeter / u.year
model_length = 100. * u.kilometer
bodyforce = 3000 * u.kilogram / u.metre**3 * 9.81 * u.meter / u.second**2
baseModelTemp = 1273.15 * u.degK
surfaceTemp = 273.15 * u.degK

KL = model_length
Kt = KL / velocity
KM = bodyforce * KL**2 * Kt**2
KT = (baseModelTemp - surfaceTemp)

GEO.scaling_coefficients["[length]"] = KL # length
GEO.scaling_coefficients["[time]"] = Kt # time
GEO.scaling_coefficients["[mass]"]= KM # mass
GEO.scaling_coefficients["[temperature]"] = KT # temperature

        
# 3. Instantiate the Model, set mesh and finite element
Model = GEO.Model(elementRes=(64, 32), 
                  minCoord=(0. * u.kilometer, -45 * u.kilometer), 
                  maxCoord=(100. * u.kilometer, 5 * u.kilometer), 
                  gravity=(0.0, -9.81 * u.meter / u.second**2))

# Model outputs will be written in the following folder:
Model.outputDir="result_model1"

# 4. Define materials' geometry (swarm)
air = Model.add_material(name="Air", shape=GEO.shapes.Layer2D(top=Model.top, bottom=Model.bottom))
crust = Model.add_material(name="Crust", shape=GEO.shapes.Layer2D(top=-0*u.kilometer, bottom=Model.bottom))

# 5. Define materials'mechanical and thermal properties
air.density    = 1. * u.kilogram / u.metre**3
crust.density  = GEO.LinearDensity(reference_density= 2700. * u.kilogram / u.metre**3,
                                        thermalExpansivity= 2.8e-5 * u.kelvin**-1)

# thermal diffusivity, heat capacity, and radiogenic heat production:
Model.diffusivity = 9e-7 * u.metre**2 / u.second 
Model.capacity    = 1000. * u.joule / (u.kelvin * u.kilogram)
crust.radiogenicHeatProd = 7.67e-7 * u.watt / u.meter**3

# rheologies: viscosity and plasticity:
rh = GEO.ViscousCreepRegistry()  # load viscous rheology registry
air.viscosity    = 1.0e18  * u.pascal * u.second # constant value
crust.viscosity  = 0.1 * rh.Wet_Olivine_Dislocation_Goetze_et_al_1978 # loaded from registry

crust.plasticity = GEO.DruckerPrager(name="Continental Crust",
                                                cohesion=15. * u.megapascal,
                                                cohesionAfterSoftening=3. * u.megapascal,
                                                frictionCoefficient=0.44,
                                                frictionAfterSoftening=0.088,
                                                epsilon1=0.01, epsilon2=0.5)


# 6. Define thermal and velocity boundary conditions
Model.set_velocityBCs(left=[-1.0 * u.centimeter / u.years, None],
                      right=[1.0 * u.centimeter / u.years, None],
                      bottom=[None, 0.0]);

Model.set_temperatureBCs(top=293.15 * u.degK, 
                         bottom=873.15 * u.degK,
                         materials=[(air, 293.15 * u.degK)]);



# 7. run the model

Model.minViscosity = 1.0e18 * u.pascal * u.second# model minimum viscosity threshold 
Model.maxViscosity = 1.0e23 * u.pascal * u.second# model maximum viscosity threshold 

# now initialise the model at t=0 for T and P 
Model.init_model(temperature="steady-state", pressure="lithostatic")

# solve the model 
Model.solver.set_inner_method("mumps")
Model.solver.set_penalty(1e6)
Model.run_for(1.0* u.megayears, checkpoint_interval=100.0* u.kiloyears)

// Software Stack

Tools you will learn

A curated open-source stack covering the full numerical modelling workflow — from containerised execution to 3D scientific visualisation.

🐳
Docker Desktop
Containerise your UW2 environment for reproducible, cross-platform execution. Removes installation pain for MPI and PETSc dependencies.
docs.docker.com/desktop ↗
🌍
Underworld2 (UW2)
Python-friendly particle-in-cell finite element code for 2D/3D geodynamics modelling. Scales from laptop to HPC. Built in Australia.
underworld2.readthedocs.io ↗
🎨
ParaView
Industry-standard open-source 3D scientific visualisation. Reads XDMF/HDF5 outputs directly; supports time-series animation and parallel rendering.
paraview.org/download ↗
🔬
ImageJ / Fiji
Powerful image analysis for quantifying deformation markers, particle trajectories, and strain localisation patterns in model snapshots.
imagej.net/ij/download ↗
🐍
Python 3 (post-processing)
Use h5py to read checkpoints, NumPy for diagnostics (Vrms, Nusselt), and matplotlib for publication-ready figures. Jupyter notebook workflow.
jupyter.org ↗
👁️
LavaVu (inline viz)
Lightweight real-time visualisation embedded in UW2 notebooks. Use it to inspect model state during development without leaving Jupyter.
github.com/OKaluza/LavaVu ↗
